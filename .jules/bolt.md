## 2025-02-28 - Optimize hashing and regex evaluation
**Learning:** For repeated generation of deterministic hashes using values from loops, running `hashlib.sha1().update()` and `.encode()` dynamically on string interpolations inside a loop is slower than constructing the full string first (e.g. via `"".join()`) and then running `.encode()` and `.update()` exactly once. For large file parsing utilizing `re.search`, checking for deterministic explicit substrings via the native Python `in` operator string fast-path provides substantial reductions in processing overhead compared to directly applying `.search()` on every line.
**Action:** Use string joins on collections prior to hashing. Pre-empt heavy regular expressions with fast literal subset string searches.
